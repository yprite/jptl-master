# 프로젝트 요구사항

이 문서는 프로젝트 아이디어, 범위, 요구사항에 대한 논의를 기록합니다.

## 프로젝트 상태

### 현재 상황 (2024-12-27)
- **프로젝트 변경**: 기존 "블로그 시스템"에서 "JLPT 자격 검증 프로그램"으로 방향 전환
- **이유**: 사용자 요구사항에 기반한 정확한 프로젝트 정의 필요
- **진행 방식**: 규칙 준수하여 충분한 논의 후 개발 진행

## 프로젝트 개요

### JLPT 자격 검증 프로그램
**프로젝트명**: JLPT Skill Assessment Platform
**목적**: 일본어 학습자를 위한 JLPT 자격 검증 및 실력 향상 지원 플랫폼
**대상 사용자**: JLPT 시험 준비생, 일본어 학습자, 교육 기관

## 핵심 기능 요구사항

### 1. 사용자 관리
- 사용자 등록/로그인
- 학습 프로필 관리
- JLPT 목표 레벨 설정
- 학습 이력 추적

### 2. JLPT 진단 및 평가
- **레벨별 진단 테스트**: N1~N5 각 레벨별 실력 평가
- **문제 유형별 평가**: 어휘/문법/독해/청해 각 영역별 분석
- **실시간 피드백**: 정답/오답 표시 및 해설 제공
- **진단 리포트**: 강점/약점 분석 및 개선 방향 제시

### 3. 학습 콘텐츠 제공
- **문제 은행**: 레벨별, 유형별 문제 데이터베이스
- **학습 추천 시스템**: 약점 영역에 대한 맞춤형 문제 추천
- **진행도 추적**: 학습량, 정확도, 시간 추이 분석

### 4. 모의 시험 기능
- **실전 모의고사**: 실제 시험과 유사한 환경 제공
- **시간 제한**: 실제 시험 시간 준수
- **결과 분석**: 점수, 순위, 영역별 성취도

## 기술 스택

### 백엔드
- **프레임워크**: FastAPI (Python)
- **데이터베이스**: PostgreSQL
- **ORM**: SQLAlchemy
- **인증**: JWT 토큰
- **API 문서화**: OpenAPI/Swagger

### 프론트엔드
- **프레임워크**: React + TypeScript
- **상태 관리**: Zustand 또는 Redux Toolkit
- **스타일링**: Tailwind CSS
- **테스트**: Jest + React Testing Library

### 인프라
- **배포**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **모니터링**: Prometheus + Grafana

## 프로젝트 범위 (MVP)

### Phase 1: 핵심 기능 (현재 스프린트)
1. **사용자 관리 시스템**: 기본 회원가입/로그인
2. **JLPT N5 진단 테스트**: 기본적인 진단 기능
3. **결과 표시**: 점수 및 간단한 분석

### Phase 2: 확장 기능
1. **모든 JLPT 레벨 지원**: N1~N5 완전 지원
2. **학습 추천 시스템**: AI 기반 맞춤 추천
3. **모의 시험 기능**: 실전 감각 훈련

### Phase 3: 고급 기능
1. **학습 커뮤니티**: 사용자 간 정보 공유
2. **프로그레시브 웹 앱**: 오프라인 학습 지원
3. **모바일 앱**: React Native 기반 네이티브 앱
