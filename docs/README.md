# 프로젝트 문서

이 폴더는 프로젝트의 모든 문서화를 담당합니다. 코드 변경 시마다 관련 문서를 즉시 업데이트하고, 새로운 기능 추가 시 해당 문서를 생성합니다.

## 문서 구조

- `architecture/` - 시스템 아키텍처 및 설계 문서
- `api/` - API 엔드포인트 및 스키마 문서
- `development/` - 개발 환경 및 배포 관련 문서
- `user-guides/` - 사용자 가이드 및 튜토리얼
- `CHANGELOG.md` - 프로젝트 변경 이력

## 문서화 원칙

### 지속적 업데이트
- 코드 변경 즉시 관련 문서 업데이트
- 새로운 기능 추가 시 문서 생성
- 문서 리팩토링 필요시 즉시 수행

### 품질 유지
- 모든 문서는 명확하고 이해하기 쉽게 작성
- 코드와 문서의 일관성 유지
- 팀원 모두가 접근 가능한 언어 사용

## 시작하기

1. [개발 환경 설정](development/setup.md) 문서 확인
2. [아키텍처 개요](architecture/overview.md)로 시스템 이해
3. [API 문서](api/)로 엔드포인트 확인
