# JLPT 자격 검증 프로그램 제품 백로그

이 문서는 JLPT 자격 검증 프로그램의 개발을 위해 계획된 모든 기능과 작업을 우선순위별로 정리합니다.

## 백로그 관리 원칙

### 우선순위 결정 기준 (MoSCoW)
- **Must Have**: MVP 필수 기능 (P0)
- **Should Have**: 중요한 기능 (P1)
- **Could Have**: 있으면 좋은 기능 (P2)
- **Won't Have**: 이번 버전 제외 (P3)

### 백로그 정제 (Refinement)
- **주기**: 매 스프린트 시작 전
- **목적**: 백로그 항목 상세화, 우선순위 조정, 스토리 포인트 추정
- **참여자**: 개발자 1인 (현재)

### 📝 사용자 스토리 (User Stories)

#### JLPT 학습자 관점 스토리

##### US-JLPT-001: 학습자 등록 및 목표 설정
**우선순위**: P0 | **상태**: ✅ Done | **스토리 포인트**: 3
**As a**: JLPT 준비생
**I want**: 플랫폼에 가입하고 목표 레벨을 설정할 수 있다
**So that**: 나의 학습 목표를 명확히 하고 진도 추적이 가능하다
**수용 기준**:
- 이메일과 사용자명으로 가입 가능
- N5-N1 중 목표 레벨 선택 가능
- 가입 후 즉시 목표 레벨 설정됨

##### US-JLPT-002: N5 진단 테스트 응시
**우선순위**: P0 | **상태**: 📋 Ready | **스토리 포인트**: 8
**As a**: N5 레벨 준비생
**I want**: N5 진단 테스트를 응시하고 결과를 확인할 수 있다
**So that**: 현재 실력을 파악하고 개선 방향을 알 수 있다
**수용 기준**:
