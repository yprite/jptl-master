# 태스크 관리 가이드

이 문서는 프로젝트의 태스크 관리 및 스프린트 운영 방법을 정의합니다.

## 스프린트 개요

### 스프린트 주기
- **기간**: 1주 (월요일 ~ 일요일)
- **목표**: 매주 최소 1개의 기능 완성 또는 개선
- **리뷰**: 매주 금요일 스프린트 리뷰 및 회고

### 스프린트 프로세스
1. **계획 수립** (월요일): 이번 주 목표 및 태스크 선정
2. **일일 스탠드업** (매일 아침): 진행 상황 공유
3. **작업 진행**: 우선순위 기반 태스크 처리
4. **리뷰 및 회고** (금요일): 성과 평가 및 개선점 도출

## 우선순위 기준

### 우선순위 레벨
1. **🔴 Critical (P0)**: 시스템 다운, 보안 이슈, 긴급 버그
2. **🟠 High (P1)**: 핵심 기능, 사용자 경험 영향
3. **🟡 Medium (P2)**: 기능 개선, 기술 부채 해결
4. **🟢 Low (P3)**: 사소한 개선, 미래 대비 작업

### 우선순위 결정 요소
- **비즈니스 임팩트**: 사용자 수, 수익 영향
- **기술적 중요도**: 안정성, 성능, 보안
- **의존성**: 다른 작업에 미치는 영향
- **난이도**: 구현 복잡도 및 시간

## 태스크 관리 방법

### 태스크 생성
- **제목**: 명확하고 간결하게 작성
- **설명**: 무엇을, 왜 하는지 상세히 기술
- **수용 기준 (Acceptance Criteria)**: 완료 조건 명시
- **우선순위 및 스토리 포인트**: 추정 난이도 표시

### 태스크 상태
- **📋 Backlog**: 계획 중인 작업
- **⏳ In Progress**: 진행 중인 작업
- **✅ Done**: 완료된 작업
- **🚫 Blocked**: 차단된 작업 (이유 명시)

### 태스크 템플릿
```
## 🎯 태스크 제목

### 📝 설명
- 작업 목적 및 배경 설명

### ✅ 수용 기준 (Acceptance Criteria)
- [ ] 기준 1
- [ ] 기준 2
- [ ] 기준 3

### 🔗 관련 링크
- 관련 이슈 또는 문서

### 📊 우선순위 및 추정
- 우선순위: P1
- 스토리 포인트: 3
- 예상 완료일: YYYY-MM-DD

### 🧪 테스트 케이스
- 단위 테스트: [ ]
- 통합 테스트: [ ]
- E2E 테스트: [ ]
```

## 스프린트 관리

### 주간 계획 수립
1. **Backlog 검토**: 우선순위 재정렬
2. **용량 평가**: 이번 주 처리 가능한 작업량 산정
3. **태스크 할당**: 각 태스크에 담당자 지정
4. **목표 설정**: 스프린트 목표 명확히 정의

### 진행 상황 추적
- **데일리 스탠드업**: 15분 내 진행 상황 공유
- **진행률 업데이트**: 태스크 상태 실시간 반영
- **장애물 식별**: 블로킹 이슈 즉시 보고

### 스프린트 리뷰
- **성과 평가**: 완료된 작업 검토
- **시연**: 실제 동작 확인
- **회고**: 잘 된 점, 개선할 점 도출
- **다음 스프린트 계획**: 교훈 반영한 계획 수립

## 도구 및 템플릿

### 추천 도구
- **GitHub Issues**: 태스크 관리 및 추적
- **GitHub Projects**: 칸반 보드 방식 관리
- **Milestone**: 스프린트 목표 관리

### 템플릿 사용
- 모든 태스크는 표준 템플릿 사용
- 라벨로 우선순위, 타입, 상태 구분
- 마일스톤으로 스프린트 관리
