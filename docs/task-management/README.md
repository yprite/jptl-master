# AI 기반 태스크 관리 시스템

이 문서는 AI-사용자 협업을 위한 태스크 상태 저장소입니다.

## 시스템 목적
- **AI 파싱 최적화**: AI가 빠르게 현재 상태를 파악하고 사용자에게 보고
- **실시간 동기화**: 코드 변경사항과 함께 자동으로 상태 업데이트
- **질문 기반 조회**: 사용자가 질문하면 AI가 이 데이터를 기반으로 답변
- **자동화된 추적**: TDD 완료, 커밋 분석을 통한 진행상황 자동 기록

## 데이터 구조
- **current-sprint.md**: 현재 스프린트 상태 (키-값 포맷)
- **backlog.md**: 전체 백로그 상태 (구조화된 목록)
- **자동 업데이트**: AI가 코드 변경 감지 시 실시간 반영

## 우선순위 기준

### 우선순위 레벨
1. **🔴 Critical (P0)**: 시스템 다운, 보안 이슈, 긴급 버그
2. **🟠 High (P1)**: 핵심 기능, 사용자 경험 영향
3. **🟡 Medium (P2)**: 기능 개선, 기술 부채 해결
4. **🟢 Low (P3)**: 사소한 개선, 미래 대비 작업

### 우선순위 결정 요소
- **비즈니스 임팩트**: 사용자 수, 수익 영향
- **기술적 중요도**: 안정성, 성능, 보안
- **의존성**: 다른 작업에 미치는 영향
- **난이도**: 구현 복잡도 및 시간

## 태스크 관리 방법

### 태스크 생성
- **제목**: 명확하고 간결하게 작성
- **설명**: 무엇을, 왜 하는지 상세히 기술
- **수용 기준 (Acceptance Criteria)**: 완료 조건 명시
- **우선순위 및 스토리 포인트**: 추정 난이도 표시

### 태스크 상태
- **📋 Backlog**: 계획 중인 작업
- **⏳ In Progress**: 진행 중인 작업
- **✅ Done**: 완료된 작업
- **🚫 Blocked**: 차단된 작업 (이유 명시)

### 태스크 템플릿
```
## 🎯 태스크 제목

### 📝 설명
- 작업 목적 및 배경 설명

### ✅ 수용 기준 (Acceptance Criteria)
- [ ] 기준 1
- [ ] 기준 2
- [ ] 기준 3

### 🔗 관련 링크
- 관련 이슈 또는 문서

### 📊 우선순위 및 추정
- 우선순위: P1
- 스토리 포인트: 3
- 예상 완료일: YYYY-MM-DD

### 🧪 테스트 케이스
- 단위 테스트: [ ]
- 통합 테스트: [ ]
- E2E 테스트: [ ]
```

## 스프린트 관리

### 주간 계획 수립
1. **Backlog 검토**: 우선순위 재정렬
2. **용량 평가**: 이번 주 처리 가능한 작업량 산정
3. **태스크 할당**: 각 태스크에 담당자 지정
4. **목표 설정**: 스프린트 목표 명확히 정의

### 진행 상황 추적
- **데일리 스탠드업**: 15분 내 진행 상황 공유
- **진행률 업데이트**: 태스크 상태 실시간 반영
- **장애물 식별**: 블로킹 이슈 즉시 보고

### 스프린트 리뷰
- **성과 평가**: 완료된 작업 검토
- **시연**: 실제 동작 확인
- **회고**: 잘 된 점, 개선할 점 도출
- **다음 스프린트 계획**: 교훈 반영한 계획 수립

## AI 기반 태스크 추적

### 자동화된 상태 관리
- **실시간 동기화**: 코드 변경사항 감지 시 자동 상태 업데이트
- **TDD 연동**: 테스트 완료 시 태스크 상태 자동 변경
- **메트릭스 추적**: 커버리지, 복잡도, 변경 빈도 자동 기록

### AI 파싱 최적화 템플릿
- **키-값 포맷**: AI가 빠르게 파싱할 수 있는 구조화된 데이터
- **메타데이터 우선**: 파일 상단에 핵심 정보 배치
- **자동 태그**: AI가 인식할 수 있는 특수 태그 사용
